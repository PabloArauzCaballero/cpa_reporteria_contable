# Principios aplicados

Este proyecto se generó bajo los principios del repo CPA original:

- API fuera del JSX.
- Tipos de dominio separados de DTOs.
- Mappers explícitos para no acoplar la UI al backend.
- Cálculos financieros fuera de componentes visuales.
- Componentes pequeños y especializados.
- Variables CSS derivadas de la paleta CPA.
- Sin dependencias innecesarias.
- Sin CRUD; módulo estrictamente de reportería.
- Estados loading, error, vacío y datos.
- Mock opcional para arrancar sin backend.
